🧠 Core Concepts

1. What Is an Agent?

An AI Agent is an LLM that can:
1. Observe — receive input (text, data, tool results)
2. Reason — decide what action to take next
3. Act — call a tool, another agent, or produce output
4. Iterate — repeat until a goal is complete

Observe → Think → Act → Observe → ...

Analogy: An agent is like a developer with access to Google, a calculator, and a database. They read the problem, decide how to approach it, use their tools, and keep going until the task is done.

2. The ReAct Pattern (Reason + Act)

The foundational reasoning loop behind most agents:

Thought: "I need to find the population of India"
Action: search("India population 2024")
Observation: "India population is 1.44 billion"
Thought: "Now I can answer the question"
Final Answer: "India's population is 1.44 billion"

This is implemented by prompting the LLM with a structured system prompt that includes available tools and expecting it to produce Thought → Action → Observation format.

3. What Is a Multi-Agent System?

A Multi-Agent System (MAS) is a collection of agents that each have:
- A specific role (specialization)
- Individual memory (local context)
- Shared state (global context)
- Communication protocols (how they pass information)

Mental Model: The Restaurant Kitchen

Role Kitchen Equivalent Agent Role
Sous Chef Task coordinator Planner Agent
Line Cook Domain expert Researcher Agent
Quality Control Fact checker Critic Agent
Head Chef Final authority Supervisor Agent

Each role is specialized and passes work to the next stage. No single person does everything.

4. Agent Coordination Patterns

Pattern A: Sequential (Pipeline)

Agent1 → Agent2 → Agent3 → Output

Simple but rigid. Each agent depends on the previous one.

Pattern B: Hierarchical (Supervisor-Worker)

        Supervisor
       /     |     \
  Agent1  Agent2  Agent3

Supervisor decides which worker to call. Workers report back. More flexible.

Pattern C: Graph-Based (LangGraph)

         ┌─────────┐
         │ Planner │
         └────┬────┘
         ┌────▼────┐   loop   ┌──────────┐
         │Research │◄─────────│  Critic  │
         └────┬────┘          └──────────┘
         ┌────▼────┐
         │ Writer  │
         └─────────┘

Nodes are agents, edges are transitions. Conditional edges allow loops, branching, and early exits. This is the most powerful pattern.

5. LangGraph Mental Model

LangGraph models agent workflows as a state machine:

Concept Description Code Equivalent
Node An agent or function def planner_node(state)
Edge Transition between nodes graph.add_edge("planner", "researcher")
Conditional Edge Branch based on state graph.add_conditional_edges(...)
State Shared data between nodes class GraphState(TypedDict)
Entrypoint Where execution starts graph.set_entry_point("planner")
END Terminal node graph.add_edge("writer", END)

Key Insight: LangGraph is just a directed graph where nodes are Python functions and edges pass a shared state object. If you can draw the flowchart, you can implement it in LangGraph.

6. Shared State (TypedDict)

All agents read from and write to a single shared state object:

from typing import TypedDict, List, Optional

class GraphState(TypedDict):
    query: str                    # Original user query
    plan: List[str]               # Subtasks from Planner
    research_results: List[dict]  # Findings from Researcher
    analysis: str                 # Synthesis from Analyst
    final_report: str             # Final output from Writer
    iteration_count: int          # Loop guard
    approved: bool                # Human-in-the-loop flag

Each node receives the full state and returns only the updated fields. This is like a Redux store for your agent pipeline.

9. Common Misconceptions

❌ "More agents = better results"

Reality: More agents = more latency, cost, and debugging complexity. Use only as many agents as the task demands.

❌ "Agents always work reliably"

Reality: LLMs are probabilistic. Always add validation, retries, and fallbacks.

❌ "Chains and graphs are the same"

Reality: Chains are sequential and fixed. Graphs support loops, conditions, and parallelism.

❌ "You need agents for everything"

Reality: Simple retrieval tasks (FAQ bot, single lookup) are better served by chains or direct API calls.
