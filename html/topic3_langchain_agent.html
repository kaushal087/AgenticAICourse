🦜 Build Your First Agent with Python: LangChain

Topic 3 LangChain Tools Memory Agents

Prerequisites: Python intermediate · Basic LLM familiarity
Outcome: Build a fully functional, deployable LangChain agent with tools and memory

"LangChain is the scaffolding that transforms a raw LLM into a capable software agent. Understand its architecture and you understand how modern AI agents are built."

🎯 Learning Objectives

  • Map out LangChain's architecture: LLMs, Prompts, Chains, Tools, Memory, Agents
  • Build a modular agent capable of reasoning, tool use, and decision-making
  • Implement tool invocation, function calling, and conversational memory
  • Wire everything into a functional, deployable Python agent

1. LangChain Architecture Overview

The Six Core Abstractions

AbstractionWhat it isAnalogy
LLM / ChatModelThe reasoning engine. Wraps OpenAI, Anthropic, etc.The brain
PromptTemplateReusable, parameterized prompt structureA fill-in-the-blank form
ChainSequence of components composed togetherAssembly line
ToolPython function the agent can choose to callA specialized skill
MemoryState persisted across LLM callsNotepad
AgentLLM-driven loop that selects tools and actionsThe decision-maker

How They Fit Together

User Input
    │
    ▼
PromptTemplate ──► formats input + memory + tools description
    │
    ▼
ChatModel (LLM) ──► generates Thought + Action
    │
    ├─► Tool Call? ──► Tool.run(input) ──► Observation ──► back to LLM
    │
    └─► Final Answer? ──► Response to user
         │
         ▼
       Memory.save_context(input, output)

2. LLMs and Prompts in LangChain

from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.messages import HumanMessage, SystemMessage

# Initialize the LLM
llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0,          # Deterministic for agents
    max_tokens=1000,
    timeout=30,
)

# Basic prompt template
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant specializing in {domain}. Be concise and technical."),
    MessagesPlaceholder(variable_name="chat_history"),  # Memory slot
    ("human", "{input}"),
    MessagesPlaceholder(variable_name="agent_scratchpad"),  # Agent reasoning slot
])

# Invoke directly
messages = prompt.format_messages(
    domain="software engineering",
    chat_history=[],
    input="What is a context manager in Python?",
    agent_scratchpad=[]
)
response = llm.invoke(messages)
print(response.content)

3. Defining Tools

Three Ways to Define a Tool

Method 1: @tool decorator (simplest)

from langchain.tools import tool

@tool
def calculate(expression: str) -> str:
    """Evaluate a mathematical expression. Input must be a valid Python expression.
    Example: '(15 * 24) / 3 + 100'"""
    try:
        # Use ast.literal_eval for safety — never eval() raw LLM output
        import ast
        tree = ast.parse(expression, mode='eval')
        # Only allow safe math operations
        allowed_nodes = (ast.Expression, ast.BinOp, ast.UnaryOp, ast.Num,
                        ast.Add, ast.Sub, ast.Mult, ast.Div, ast.Pow,
                        ast.USub, ast.Constant)
        for node in ast.walk(tree):
            if not isinstance(node, allowed_nodes):
                return "Error: Only arithmetic expressions allowed."
        result = eval(compile(tree, '', 'eval'))  # noqa
        return str(result)
    except Exception as e:
        return f"Calculation error: {str(e)}"

@tool
def get_current_date(_: str = "") -> str:
    """Get the current date and time."""
    from datetime import datetime
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S UTC")

@tool
def search_wikipedia(query: str) -> str:
    """Search Wikipedia for factual information about a topic."""
    import wikipedia
    try:
        return wikipedia.summary(query, sentences=3)
    except Exception:
        return f"No Wikipedia article found for: {query}"

Method 2: StructuredTool with Pydantic schema

from langchain.tools import StructuredTool
from pydantic import BaseModel, Field

class WeatherInput(BaseModel):
    city: str = Field(description="The city name to get weather for")
    units: str = Field(default="celsius", description="Temperature units: 'celsius' or 'fahrenheit'")

def get_weather(city: str, units: str = "celsius") -> str:
    """Get real weather data via API."""
    # In production, call a real weather API
    return f"Weather in {city}: 22°{'C' if units == 'celsius' else 'F'}, partly cloudy"

weather_tool = StructuredTool.from_function(
    func=get_weather,
    name="get_weather",
    description="Get the current weather for a city.",
    args_schema=WeatherInput
)

4. Conversational Memory

from langchain.memory import ConversationBufferWindowMemory

# Keep only the last N exchanges (prevents unbounded context growth)
memory = ConversationBufferWindowMemory(
    k=5,                      # Keep last 5 turns
    return_messages=True,     # Return Message objects, not strings
    memory_key="chat_history", # Must match MessagesPlaceholder name in prompt
    output_key="output"
)

Persistent Memory (Redis)

from langchain_community.chat_message_histories import RedisChatMessageHistory
from langchain.memory import ConversationBufferMemory

def get_session_memory(session_id: str) -> ConversationBufferMemory:
    """Load or create persistent memory for a user session."""
    history = RedisChatMessageHistory(
        session_id=session_id,
        url="redis://localhost:6379",
        ttl=3600  # 1 hour TTL
    )
    return ConversationBufferMemory(
        chat_memory=history,
        return_messages=True,
        memory_key="chat_history",
        output_key="output"
    )

5. Building the Complete Agent

Function Calling Agent (Recommended for Production)

from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

# --- Tools ---
from langchain_community.tools.tavily_search import TavilySearchResults
tools = [
    TavilySearchResults(max_results=3),
    calculate,
    get_current_date,
    weather_tool
]

# --- LLM ---
llm = ChatOpenAI(model="gpt-4o", temperature=0)

# --- Prompt ---
prompt = ChatPromptTemplate.from_messages([
    ("system", """You are a helpful, precise assistant. Use your tools when you need
real-time information or calculations. Always cite your sources."""),
    MessagesPlaceholder("chat_history"),
    ("human", "{input}"),
    MessagesPlaceholder("agent_scratchpad"),
])

# --- Memory ---
memory = ConversationBufferWindowMemory(
    k=10, return_messages=True,
    memory_key="chat_history", output_key="output"
)

# --- Agent ---
agent = create_openai_functions_agent(llm=llm, tools=tools, prompt=prompt)
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    memory=memory,
    verbose=True,
    max_iterations=6,
    handle_parsing_errors=True,   # Don't crash on malformed tool calls
    return_intermediate_steps=True
)

# --- Run ---
def chat(user_message: str) -> str:
    result = executor.invoke({"input": user_message})
    return result["output"]

print(chat("What's the weather in London, and what's 15% of 847?"))
print(chat("Wait, I meant Paris. Also, what date did we first start chatting?"))
# Memory ensures the agent recalls the context of previous messages

6. The LCEL Way: LangChain Expression Language

Modern LangChain uses LCEL (| pipe operator) to compose chains declaratively:

from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

# Simple chain: prompt | llm | parser
simple_chain = (
    ChatPromptTemplate.from_template("Explain {concept} in one sentence.")
    | ChatOpenAI(model="gpt-4o-mini")
    | StrOutputParser()
)

result = simple_chain.invoke({"concept": "vector embeddings"})
print(result)

# Agents as LCEL runnables
from langchain.agents import AgentExecutor
from langchain_core.runnables import RunnableLambda

agent_chain = (
    RunnablePassthrough.assign(
        agent_scratchpad=lambda x: format_to_openai_function_messages(x["intermediate_steps"])
    )
    | prompt
    | llm.bind_tools(tools)
    | OpenAIFunctionsAgentOutputParser()
)

7. Full Deployable Agent (Single File)

# agent_app.py — copy-paste ready
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain.memory import ConversationBufferWindowMemory
from langchain.tools import tool
from langchain_community.tools.tavily_search import TavilySearchResults

load_dotenv()  # Load OPENAI_API_KEY and TAVILY_API_KEY from .env

@tool
def calculate(expression: str) -> str:
    """Evaluate a safe arithmetic expression like '15 * 24 + 100'."""
    import ast
    try:
        tree = ast.parse(expression, mode='eval')
        return str(eval(compile(tree, '', 'eval')))  # noqa
    except Exception as e:
        return f"Error: {e}"

tools = [TavilySearchResults(max_results=3), calculate]

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful research assistant. Use tools for real-time info and math."),
    MessagesPlaceholder("chat_history"),
    ("human", "{input}"),
    MessagesPlaceholder("agent_scratchpad"),
])
memory = ConversationBufferWindowMemory(
    k=10, return_messages=True, memory_key="chat_history", output_key="output"
)
agent = create_openai_functions_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, memory=memory, verbose=True)

if __name__ == "__main__":
    print("Agent ready. Type 'quit' to exit.")
    while True:
        user_input = input("You: ").strip()
        if user_input.lower() == "quit":
            break
        response = executor.invoke({"input": user_input})
        print(f"Agent: {response['output']}\n")

8. LangChain vs. Raw OpenAI SDK

AspectRaw OpenAI SDKLangChain
BoilerplateMore code for tool loops, memory, retriesHandled by framework
FlexibilityFull controlAbstractions hide details (can be a drawback)
Model switchingManual refactorSwap one line: ChatAnthropic()
Debug visibilityEverything is explicitUse verbose=True or LangSmith
Best forSimple, performance-critical appsComplex agent workflows, rapid prototyping
