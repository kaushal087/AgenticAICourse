🎬 Demo Walkthrough

What the Demo Does

The demo runs a Multi-Agent Research Assistant that answers complex research questions by:

  1. Planning — dividing the question into subtasks
  2. Researching — retrieving relevant information per subtask
  3. Analyzing — evaluating and synthesizing findings
  4. Writing — producing a structured final report

Demo Modes

Option A: CLI Demo (Quick)

cd code/
python main.py
  • Prints each agent's output as it executes
  • Shows intermediate state at each node
  • Fast way to see the pipeline in action

Option B: Streamlit Demo (Interactive)

cd demo/
bash run_demo.sh
  • Web UI with a text input box
  • Real-time streaming of agent progress
  • Expandable sections for each agent's output
  • Final report rendered as formatted Markdown

Component Interaction Map

demo_app.py (Streamlit)
    │
    │ user_query (str)
    ▼
code/main.py
    │
    │ builds initial GraphState
    ▼
LangGraph app.stream()
    │
    ├──▶ agent.py::planner_node()
    │       Uses: OpenAI GPT-4o
    │       Reads: state["query"]
    │       Writes: state["plan"]
    │
    ├──▶ agent.py::researcher_node()
    │       Uses: OpenAI GPT-4o + tools
    │       Calls: tools.py::search_web()
    │       Calls: rag_pipeline.py::retrieve()
    │       Reads: state["plan"]
    │       Writes: state["research_results"]
    │
    ├──▶ agent.py::analyst_node()
    │       Uses: OpenAI GPT-4o
    │       Reads: state["research_results"]
    │       Writes: state["analysis"], state["approved"]
    │
    └──▶ agent.py::writer_node()
            Uses: OpenAI GPT-4o
            Reads: state["query"], state["analysis"], state["research_results"]
            Writes: state["final_report"]

Running the Demo Step by Step

Step 1: Setup

# Clone / navigate to project
cd multi-agent-langgraph/

# Install dependencies
pip install -r code/requirements.txt

# Set OpenAI API key
export OPENAI_API_KEY="sk-your-key-here"

Step 2: Load Sample Data into RAG

The demo includes a sample knowledge base in demo/sample_data/knowledge_base.txt.
It gets automatically loaded and indexed when the demo starts.

# This happens automatically in main.py
rag = RAGPipeline()
rag.ingest("demo/sample_data/knowledge_base.txt")

Key Code Files Explained

code/agent.py

Contains all four agent node functions. Each function:
- Receives GraphState
- Builds a prompt for the LLM
- Calls GPT-4o (with or without tools)
- Returns a dict with updated state fields

Most important pattern: Agent functions are just regular Python functions:

def planner_node(state: GraphState) -> dict:
    # Build prompt using state["query"]
    # Call LLM
    # Parse response into plan
    return {"plan": [...]}  # Only return what changed

code/tools.py

Defines tools that agents can call:
- search_web(query: str) → str — Tavily web search
- calculate(expression: str) → str — Safe math evaluation
- rag_retrieve(query: str) → str — Local document retrieval

code/rag_pipeline.py

Manages the vector store:
- ingest(file_path) — load and index documents
- retrieve(query) — find top-K relevant chunks
- Uses FAISS for vector similarity search
- Uses text-embedding-3-small for embeddings

demo/demo_app.py

Streamlit app that:
- Provides a text input for the query
- Shows a progress indicator while agents run
- Streams and displays each agent's output in expandable sections
- Renders the final report with Markdown formatting

Instructor Talking Points

During the demo, say:

  1. "Watch how the Planner breaks down one complex question — this is the key insight. Humans do this too, but we formalize it for AI."

  2. "Notice the Researcher is calling TWO tools per subtask. This is parallel information gathering — like a developer using both Google and internal docs."

  3. "The Analyst node is the most important for reliability. Without a validation step, you'd ship whatever the Researcher returned, even if it was incomplete."

  4. "The loop — if analyst is not satisfied, we go back to Researcher — this is what makes the system self-correcting. That's the difference between a chatbot and an agent."

  5. "The final report cites sources. In production, this is critical for trust and auditability of AI outputs."
