📊 Evaluating & Optimizing Agents with LangSmith

Topic 8 LangSmith Evaluation Observability Optimization

Prerequisites: Topics 1–4 · LangChain basics
Outcome: Instrument, evaluate, and continuously optimize a production agent with LangSmith

"You can't improve what you can't measure. Agent evaluation is what separates a demo from a production system."

🎯 Learning Objectives

  • Explain why agent evaluation is harder than traditional software testing
  • Set up LangSmith tracing and interpret execution traces
  • Define and implement evaluation metrics for agents
  • Optimize inference latency, model selection, and token costs
  • Compare fine-tuning vs. prompt optimization for improved agent performance
  • Deploy a continuous evaluation pipeline in production

1. Why Agent Evaluation Is Hard

Traditional software testing: inputs are deterministic, outputs are binary (pass/fail), test suites run in milliseconds.

Agent evaluation: outputs are stochastic, correctness is often subjective, evaluation itself may require an LLM, and a single run can take 30+ seconds and cost $0.50.

ChallengeTraditional SoftwareLLM Agents
Output determinism✅ Always same output❌ Varies per run (temperature > 0)
Correctness definitionExact match / assertSemantic correctness (needs human or LLM judge)
Failure modesExceptions, wrong valuesHallucination, wrong tool, infinite loops, unhelpful but valid output
Test costMilliseconds, freeSeconds to minutes, $0.01–$1 per run
Multi-step complexityTest each functionMust evaluate entire reasoning chain

What to Measure

Metric CategorySpecific Metrics
CorrectnessAnswer accuracy, factual grounding (citations), task completion rate
FaithfulnessIs the answer supported by retrieved context? No hallucinated facts?
RelevanceIs the answer relevant to the question? No off-topic drift?
EfficiencyLatency (p50, p95, p99), token usage, number of LLM calls, tool call count
SafetyGuardrail bypass rate, PII leakage rate, refusal accuracy
User satisfactionThumbs up/down, conversation abandonment rate

2. LangSmith: Setup and Tracing

What LangSmith Does

LangSmith is LangChain's observability and evaluation platform. It automatically captures every LLM call, tool invocation, and chain step with full inputs, outputs, latencies, and token counts — all in a searchable UI.

Setup (5 lines)

# 1. Install
pip install langsmith

# 2. Set environment variables
export LANGCHAIN_TRACING_V2=true
export LANGCHAIN_API_KEY="ls_..."       # Get from smith.langchain.com
export LANGCHAIN_PROJECT="my-agent-v1"  # Groups traces by project

# That's it. LangChain automatically sends all traces to LangSmith.
# Run your agent normally:
result = agent.invoke({"input": "What is the capital of France?"})
# → Trace automatically appears at smith.langchain.com

Adding Custom Spans

from langsmith import traceable

@traceable(name="rag_retrieval", tags=["retrieval"])
def retrieve_documents(query: str) -> list[str]:
    """Custom trace span for RAG retrieval."""
    docs = vectorstore.similarity_search(query, k=4)
    return [d.page_content for d in docs]

@traceable(name="full_pipeline", metadata={"version": "1.2.0"})
def run_pipeline(question: str) -> str:
    docs = retrieve_documents(question)   # Auto-traced as child span
    context = "\n".join(docs)
    response = llm.invoke(f"Context: {context}\n\nQuestion: {question}")
    return response.content

# Run with trace
answer = run_pipeline("What is our refund policy?")

3. Programmatic Evaluation with LangSmith

Creating a Dataset

from langsmith import Client

client = Client()

# Create evaluation dataset
dataset = client.create_dataset(
    dataset_name="refund-policy-qa-v1",
    description="Questions about our refund and return policies"
)

# Add examples (question → expected answer)
examples = [
    ("How long is your return window?", "30 days from purchase"),
    ("Can I return a used item?", "Items must be in original condition"),
    ("How do I initiate a refund?", "Contact support@acme.com or use the online portal"),
]

for question, expected in examples:
    client.create_example(
        inputs={"input": question},
        outputs={"output": expected},
        dataset_id=dataset.id
    )

Running an Evaluation

from langsmith.evaluation import evaluate, LangChainStringEvaluator

# Define the agent function to evaluate
def agent_under_test(inputs: dict) -> dict:
    result = executor.invoke({"input": inputs["input"], "chat_history": []})
    return {"output": result["output"]}

# Define evaluators
evaluators = [
    # Correctness: LLM judges if agent answer matches reference answer
    LangChainStringEvaluator("labeled_criteria", config={
        "criteria": "correctness",
        "llm": ChatOpenAI(model="gpt-4o", temperature=0)
    }),
    # Conciseness
    LangChainStringEvaluator("criteria", config={
        "criteria": "conciseness",
        "llm": ChatOpenAI(model="gpt-4o", temperature=0)
    }),
]

# Run evaluation
results = evaluate(
    agent_under_test,
    data="refund-policy-qa-v1",
    evaluators=evaluators,
    experiment_prefix="agent-v1-baseline",
    max_concurrency=3
)

print(f"Correctness: {results.summary_metrics['correctness/score']:.2f}")
print(f"Conciseness: {results.summary_metrics['conciseness/score']:.2f}")

RAG-Specific Evaluation (RAGAS)

from ragas import evaluate as ragas_evaluate
from ragas.metrics import (
    faithfulness,       # Is the answer grounded in retrieved context?
    answer_relevancy,   # Is the answer relevant to the question?
    context_precision,  # Are retrieved docs actually relevant?
    context_recall      # Were all relevant docs retrieved?
)
from datasets import Dataset

eval_data = Dataset.from_dict({
    "question": ["What is our SLA?", "How do I cancel?"],
    "answer": [agent_output_1, agent_output_2],
    "contexts": [retrieved_docs_1, retrieved_docs_2],
    "ground_truth": ["99.9% uptime SLA", "Cancel via account settings or email"]
})

results = ragas_evaluate(
    eval_data,
    metrics=[faithfulness, answer_relevancy, context_precision, context_recall]
)
print(results.to_pandas())

4. Optimizing Latency, Cost, and Token Usage

Latency Optimization Strategies

StrategyTypical GainTrade-off
Use GPT-4o-mini instead of GPT-4o2–3× faster, 10× cheaperSlightly lower quality on complex tasks
Semantic caching (repeat queries)100× faster, 100% cheaperStale cache if data changes
Reduce max_iterations in agent1.5–2× fasterMay not complete complex tasks
Trim conversation memory (window=3)20–40% fewer tokensLose older context
Streaming responsesSame total latency, better UXSlightly more complex code
Async parallel tool calls2–4× faster tool executionMore complex orchestration
from langchain.globals import set_llm_cache
from langchain_community.cache import RedisSemanticCache
from langchain_openai import OpenAIEmbeddings

# Semantic cache: if a similar question was answered recently, return cached result
set_llm_cache(RedisSemanticCache(
    redis_url="redis://localhost:6379",
    embedding=OpenAIEmbeddings(),
    score_threshold=0.95   # 95% semantic similarity = cache hit
))

# Measure token usage
from langchain.callbacks import get_openai_callback

with get_openai_callback() as cb:
    result = executor.invoke({"input": "What are our shipping rates?", "chat_history": []})
    print(f"Tokens used: {cb.total_tokens}")
    print(f"Cost: ${cb.total_cost:.4f}")
    print(f"LLM calls: {cb.successful_requests}")

5. Fine-Tuning vs. Prompt Optimization

AspectPrompt OptimizationFine-Tuning
Cost to implementLow ($0)High ($100–$10,000+)
Time to iterateMinutesHours to days
Inference costStandard API pricingCan be lower with smaller model
PortabilitySwitch models easilyTied to specific model version
Custom style/formatGoodExcellent
New knowledgeUse RAG insteadBakes in, but can go stale
Best forMost use cases (try first)Tone, format, specialized vocab
Rule of thumb: Always try prompt optimization first. Fine-tuning is for when you have 1,000+ high-quality examples and prompt engineering has plateaued. Never fine-tune to inject new knowledge — use RAG for that.

DSPy: Automatic Prompt Optimization

import dspy

# DSPy 2.x uses LM() and dspy.configure() (not dspy.OpenAI / dspy.settings)
lm = dspy.LM("openai/gpt-4o-mini", max_tokens=1000)
dspy.configure(lm=lm)

class QA(dspy.Signature):
    """Answer questions about company policies accurately and concisely."""
    question = dspy.InputField()
    answer = dspy.OutputField(desc="Factual, concise answer in 1-3 sentences")

class PolicyQA(dspy.Module):
    def __init__(self):
        self.predictor = dspy.ChainOfThought(QA)
        self.retriever = dspy.Retrieve(k=3)

    def forward(self, question):
        context = self.retriever(question).passages
        return self.predictor(question=question, context=context)

# Optimize prompts automatically using training examples
from dspy.teleprompt import BootstrapFewShot

optimizer = BootstrapFewShot(metric=your_metric_function, max_bootstrapped_demos=4)
optimized_program = optimizer.compile(PolicyQA(), trainset=train_examples)
# Optimized prompts are now baked into the program

6. Continuous Evaluation in Production

import asyncio
from langsmith import Client
from datetime import datetime, timedelta

client = Client()

async def run_nightly_evaluation():
    """Run evaluation suite every night at 2 AM."""
    print(f"Starting evaluation at {datetime.now()}")

    # 1. Run full test suite
    results = evaluate(
        agent_under_test,
        data="production-qa-dataset",
        evaluators=evaluators,
        experiment_prefix=f"nightly-{datetime.now().strftime('%Y%m%d')}"
    )

    # 2. Check for regressions
    correctness = results.summary_metrics.get("correctness/score", 0)
    latency_p95 = results.summary_metrics.get("latency/p95", 999)

    alerts = []
    if correctness < 0.85:
        alerts.append(f"⚠️ Correctness dropped to {correctness:.2%} (threshold: 85%)")
    if latency_p95 > 5.0:
        alerts.append(f"⚠️ P95 latency is {latency_p95:.1f}s (threshold: 5s)")

    # 3. Send alerts if regressions detected
    if alerts:
        send_slack_alert("\n".join(alerts))
        print("ALERT:", alerts)
    else:
        print(f"✅ All checks passed. Correctness: {correctness:.2%}, P95: {latency_p95:.1f}s")

    # 4. Sample production traces for human review
    recent_runs = client.list_runs(
        project_name="production-agent",
        start_time=datetime.now() - timedelta(hours=24),
        filter='and(gt(latency, 8), eq(error, null))'  # Slow runs without errors
    )
    # Add to human review queue
    for run in list(recent_runs)[:10]:
        client.create_feedback(run.id, key="needs_review", score=0)
