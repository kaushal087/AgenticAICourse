🔬 Specialized Agents

Topic 7 Domain AI Coding Healthcare Data

Prerequisites: Topics 1–4 · Domain knowledge in target area
Outcome: Design and build domain-specific agents with high-accuracy, targeted behavior

"A general-purpose agent is a generalist. A specialized agent is an expert. In production, experts win — because they know the domain vocabulary, constraints, and failure modes."

🎯 Learning Objectives

  • Explain when to specialize an agent vs. using a general-purpose LLM
  • Design agents for key verticals: data/analytics, healthcare, software coding, customer support
  • Integrate domain-specific APIs, tools, and knowledge sources
  • Implement modular, composable architectures for focused, high-accuracy behavior
  • Deploy specialized agents that perform targeted, expert-level tasks autonomously

1. General vs. Specialized Agents

AspectGeneral Agent (GPT-4o)Specialized Agent
KnowledgeBroad but shallow in domainsDeep in target domain
Tool setGeneric (search, calculator)Domain tools (SQL, FHIR, GitHub API)
System promptGenericExpert persona + domain constraints
ModelFrontier modelFrontier model OR fine-tuned domain model
GuardrailsGeneral safetyDomain compliance (HIPAA, SOC2, etc.)
Accuracy70-80% on domain tasks90%+ with domain context

The Specialization Stack

Increasing Specialization →

Generic LLM call
    → System prompt with domain expertise
        → Domain-specific tools (FHIR API, SQL, Bloomberg)
            → RAG over domain knowledge (clinical guidelines, codebase, policies)
                → Fine-tuned domain model (optional, expensive)
                    → Human-in-the-loop for high-stakes decisions

2. Data & Analytics Agent

Capabilities

  • Translate natural language to SQL (text-to-SQL)
  • Run Python code for data analysis and visualization
  • Explain data insights in business terms
  • Detect anomalies and generate reports autonomously
from langchain_openai import ChatOpenAI
from langchain.agents import create_openai_functions_agent, AgentExecutor
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain.tools import StructuredTool
from pydantic import BaseModel
import sqlite3
import pandas as pd

DATA_SYSTEM_PROMPT = """You are a Senior Data Analyst with expertise in SQL and Python.
You have access to the 'sales' database with tables: orders, customers, products, regions.
Rules:
- Only write SELECT queries (no INSERT, UPDATE, DELETE, DROP)
- When asked for a "report", always include: summary stats, trend, and a key insight
- Format numbers with commas (e.g., 1,234,567)
- Always validate that requested columns exist before querying"""

class SQLInput(BaseModel):
    query: str

def run_sql_query(query: str) -> str:
    """Execute a read-only SQL query against the sales database."""
    # Validate: only SELECT statements
    clean_query = query.strip().upper()
    if not clean_query.startswith("SELECT"):
        return "Error: Only SELECT queries are permitted."

    # Block dangerous keywords
    forbidden = ["DROP", "DELETE", "INSERT", "UPDATE", "TRUNCATE", "ALTER", "CREATE"]
    for keyword in forbidden:
        if keyword in clean_query:
            return f"Error: '{keyword}' operation not allowed."

    conn = sqlite3.connect("sales.db")
    try:
        df = pd.read_sql_query(query, conn)
        return df.to_string(index=False, max_rows=20)
    except Exception as e:
        return f"Query error: {str(e)}"
    finally:
        conn.close()

class PythonInput(BaseModel):
    code: str

def run_analysis_code(code: str) -> str:
    """Execute Python code for data analysis. Returns stdout output."""
    import io, contextlib
    stdout = io.StringIO()
    try:
        with contextlib.redirect_stdout(stdout):
            exec(code, {"pd": pd, "print": print})  # noqa
        return stdout.getvalue() or "Code executed successfully (no output)"
    except Exception as e:
        return f"Execution error: {str(e)}"

tools = [
    StructuredTool.from_function(run_sql_query, name="sql_query",
        description="Run a SELECT SQL query on the sales database.", args_schema=SQLInput),
    StructuredTool.from_function(run_analysis_code, name="python_analysis",
        description="Run Python code for calculations and data analysis.", args_schema=PythonInput)
]

llm = ChatOpenAI(model="gpt-4o", temperature=0)
prompt = ChatPromptTemplate.from_messages([
    ("system", DATA_SYSTEM_PROMPT),
    MessagesPlaceholder("chat_history"),
    ("human", "{input}"),
    MessagesPlaceholder("agent_scratchpad"),
])

data_agent = AgentExecutor(
    agent=create_openai_functions_agent(llm, tools, prompt),
    tools=tools, verbose=True, max_iterations=8
)

result = data_agent.invoke({"input": "What were the top 5 product categories by revenue last quarter?", "chat_history": []})
print(result["output"])

3. Healthcare/Clinical Agent

⚠️ Critical: Healthcare AI agents must include human-in-the-loop for clinical decisions. The agent supports clinicians — it does not replace them. Always comply with HIPAA, HL7 FHIR, and applicable regulations.

Capabilities

  • Query FHIR patient records (read-only, consented access)
  • Check drug interactions using formulary APIs
  • Retrieve clinical guidelines (RAG over medical literature)
  • Generate structured clinical summaries for review
CLINICAL_SYSTEM_PROMPT = """You are a clinical decision support assistant.
CRITICAL RULES:
1. You ASSIST clinicians — never make autonomous clinical decisions
2. Always recommend physician review for any clinical suggestion
3. cite your source for every clinical statement
4. If unsure, say "I don't have sufficient information — consult a specialist"
5. Never disclose patient PII beyond what the querying clinician needs
6. All outputs are for informational purposes only, not medical advice"""

from langchain.tools import tool

@tool
def check_drug_interactions(drugs: str) -> str:
    """Check for drug-drug interactions.
    Input: comma-separated list of drug names, e.g. 'warfarin, aspirin, ibuprofen'"""
    drug_list = [d.strip().lower() for d in drugs.split(",")]
    # In production: call RxNorm/OpenFDA API
    # Example mock response:
    known_interactions = {
        ("warfarin", "aspirin"): "MAJOR: Increased bleeding risk. Monitor INR closely.",
        ("warfarin", "ibuprofen"): "MAJOR: NSAIDs increase anticoagulation effect. Avoid combination.",
    }
    results = []
    for i, d1 in enumerate(drug_list):
        for d2 in drug_list[i+1:]:
            key = tuple(sorted([d1, d2]))
            if key in known_interactions:
                results.append(f"⚠️ {d1} + {d2}: {known_interactions[key]}")
    return "\n".join(results) if results else "No known interactions found (always verify with pharmacist)."

@tool
def get_clinical_guideline(condition_and_question: str) -> str:
    """Search clinical guidelines for a condition.
    Input: 'condition | specific question' e.g. 'type 2 diabetes | first-line medication'"""
    # In production: RAG over UpToDate, NICE guidelines, PubMed
    return f"[Guideline search for: {condition_and_question}] — Connect to clinical knowledge RAG pipeline."

4. Software Coding Agent

Capabilities

  • Generate, review, and debug code
  • Run tests and fix failing test cases iteratively
  • Explain existing codebases via RAG over the repo
  • Create PRs and interact with GitHub
CODING_SYSTEM_PROMPT = """You are a Senior Software Engineer.
Coding standards:
- Python 3.11+, type hints always, docstrings for public functions
- Security first: never hardcode secrets, validate all inputs
- Write tests alongside code (pytest)
- Use existing patterns in the codebase before introducing new ones
When debugging: reproduce → isolate → fix → verify. Never guess."""

@tool
def run_python_tests(test_file: str) -> str:
    """Run pytest on a specific test file and return results.
    Input: relative path to test file, e.g. 'tests/test_utils.py'"""
    import subprocess, os
    if not test_file.startswith("tests/"):
        return "Error: test file must be in the tests/ directory"
    if not os.path.exists(test_file):
        return f"Error: file not found: {test_file}"
    result = subprocess.run(
        ["python", "-m", "pytest", test_file, "-v", "--tb=short"],
        capture_output=True, text=True, timeout=60, cwd="."
    )
    return result.stdout[-3000:] + result.stderr[-500:]  # Limit output

@tool
def read_file_content(filepath: str) -> str:
    """Read a source file. Input: relative file path."""
    import os
    # Prevent path traversal
    safe_path = os.path.normpath(os.path.join(".", filepath))
    if not safe_path.startswith(os.path.abspath(".")):
        return "Error: path traversal not allowed"
    try:
        with open(safe_path) as f:
            return f.read()[:5000]  # Limit to 5K chars
    except FileNotFoundError:
        return f"File not found: {filepath}"

@tool
def write_file_content(filepath_and_content: str) -> str:
    """Write content to a file. Input: 'filepath|||content'"""
    parts = filepath_and_content.split("|||", 1)
    if len(parts) != 2:
        return "Error: input must be 'filepath|||content'"
    filepath, content = parts
    import os
    safe_path = os.path.normpath(os.path.join(".", filepath))
    if not safe_path.startswith(os.path.abspath(".")):
        return "Error: path traversal not allowed"
    os.makedirs(os.path.dirname(safe_path), exist_ok=True)
    with open(safe_path, "w") as f:
        f.write(content)
    return f"File written: {filepath} ({len(content)} chars)"

5. Customer Support Agent

SUPPORT_SYSTEM_PROMPT = """You are a customer support specialist for AcmeCorp.
Your goals:
1. Resolve issues on first contact when possible
2. Show empathy before jumping to solutions
3. Escalate to human agent when: customer is very upset, legal/safety issues, request exceeds your tools
4. Never promise refunds > $50 without manager approval
5. Always confirm resolution before closing: "Does that resolve your issue?"

Tone: Warm, professional, solution-focused. Max response: 3 paragraphs."""

@tool
def lookup_order(order_id: str) -> str:
    """Look up order status and details by order ID."""
    # Validate format before querying
    import re
    if not re.match(r'^ORD-\d{6}$', order_id):
        return "Invalid order ID format. Expected: ORD-XXXXXX"
    # In production: query order management system
    return f"Order {order_id}: Shipped 2025-03-10, estimated delivery 2025-03-14. Tracking: 1Z999AA10123456784"

@tool
def process_refund(order_id_and_amount: str) -> str:
    """Process a refund. Input: 'order_id|amount_usd'"""
    parts = order_id_and_amount.split("|")
    if len(parts) != 2:
        return "Error: input must be 'order_id|amount'"
    order_id, amount_str = parts
    try:
        amount = float(amount_str)
        if amount > 50:
            return "Refund > $50 requires manager approval. Escalating to senior support."
        # In production: call payment API
        return f"Refund of ${amount:.2f} processed for {order_id}. Credit in 3-5 business days."
    except ValueError:
        return f"Invalid amount: {amount_str}"

@tool
def escalate_to_human(reason: str) -> str:
    """Escalate the conversation to a human agent with a reason."""
    return f"Escalating to human agent. Reason: {reason}. Transfer ID: ESC-{hash(reason) % 10000:04d}"

6. Modular Architecture: The Capability Router

from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
import json

# Route incoming requests to the right specialized agent
ROUTER_PROMPT = """You are a request router. Classify this request into one category:
- "data": SQL queries, analytics, reports, data exploration
- "coding": code generation, debugging, tests, code review
- "support": order issues, refunds, account problems, product questions
- "general": anything else

Request: {request}
Respond with ONLY one word from the list above."""

router_chain = (
    ChatPromptTemplate.from_template(ROUTER_PROMPT)
    | ChatOpenAI(model="gpt-4o-mini", temperature=0)
    | StrOutputParser()
)

# Agent registry
AGENTS = {
