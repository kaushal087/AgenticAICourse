📝 Quiz — Multi-Agent Systems with LangGraph

Take this quiz to test your understanding of multi-agent systems, LangGraph, and agentic design patterns.

Section A: Conceptual Questions (10 questions)

Q1. What is the primary advantage of a multi-agent system over a single LLM call for complex tasks?

  • A) Multi-agent systems are always faster
  • B) Each agent can specialize, and complex tasks can be broken into focused subtasks
  • C) Multi-agents use less tokens
  • D) LangGraph requires multiple agents to function

✅ Answer: B
Specialization and task decomposition are the core advantages. Speed and cost depend on implementation.

Q2. In LangGraph, what does a "node" represent?

  • A) A prompt template
  • B) A Python function that processes the shared state
  • C) A message in the conversation history
  • D) A vector in the embedding space

✅ Answer: B
Nodes are Python functions (typically wrapping agent logic) that receive the GraphState and return updated fields.

Q3. What is GraphState in LangGraph?

  • A) The visualization of the agent graph
  • B) A database for storing agent outputs
  • C) A TypedDict that holds all shared data passed between nodes
  • D) An OpenAI-specific data format

✅ Answer: C
GraphState is Python TypedDict — a typed dictionary that flows through every node, carrying all information the agents need.

Q4. What is a "conditional edge" in LangGraph?

  • A) An edge that only activates on weekdays
  • B) A routing function that decides which node to visit next based on the current state
  • C) An edge encrypted with a condition key
  • D) A feature exclusive to the paid LangGraph tier

✅ Answer: B
Conditional edges are routing functions. They inspect the current state and return a string key indicating the next node.

Q5. In the ReAct pattern, what does the Observation step represent?

  • A) The LLM's plan for next steps
  • B) The user's input message
  • C) The result returned by a tool after the LLM calls it
  • D) A log entry written for debugging

✅ Answer: C
Observation is the tool result. The cycle is: Thought → Action → Observation → next Thought.

Q6. Which pattern describes a workflow where a Supervisor agent decides which Worker agent to call next?

  • A) Sequential pipeline
  • B) Hierarchical multi-agent system
  • C) RAG pipeline
  • D) Chain-of-thought prompting

✅ Answer: B
A Supervisor-Worker pattern is hierarchical. The supervisor makes routing decisions dynamically based on the task state.

Q7. Why do multi-agent systems use a shared state (like GraphState) instead of direct agent-to-agent message passing?

  • A) Direct messaging is not supported by Python
  • B) OpenAI does not allow inter-agent calls
  • C) Shared state makes the system inspectable, debuggable, and supports checkpointing
  • D) It reduces the number of API calls

✅ Answer: C
With shared state, you can print/inspect the full system context at any point. Direct message passing makes debugging harder and prevents checkpointing.

Q8. What is the purpose of an Analyst (Critic) agent in a multi-agent pipeline?

  • A) To write the final report
  • B) To evaluate the quality of research and decide whether to loop back or proceed
  • C) To manage the API rate limits
  • D) To convert text to embeddings

✅ Answer: B
The Analyst/Critic provides self-correction capability. Without it, the pipeline has no quality gate and may produce incomplete results.

Q9. In RAG (Retrieval-Augmented Generation), what is the role of FAISS?

  • A) A large language model for text generation
  • B) A web search engine
  • C) An in-memory vector similarity search index
  • D) A prompt management framework

✅ Answer: C
FAISS (Facebook AI Similarity Search) stores embeddings and retrieves the most semantically similar vectors for a query.

Q10. When should you prefer LangGraph over CrewAI for building a multi-agent system?

  • A) When you want the simplest possible setup with minimal code
  • B) When you need fine-grained control over workflow logic, conditional routing, and checkpointing
  • C) When you are building a non-Python system
  • D) When your agents don't need tools

✅ Answer: B
LangGraph gives you full programmatic control. CrewAI offers faster setup through higher-level abstractions but with less flexibility.

Section B: Interview-Style Questions

IQ1. You're designing a multi-agent system that needs to handle 1000 research queries per hour. Walk through the architecture decisions you'd make to ensure reliability, cost efficiency, and scalability.

Talking points for a strong answer:
- Route simple queries to single-agent path, complex to multi-agent
- Use GPT-4o-mini for Planner and Analyst nodes; GPT-4o only for Writer
- Cache frequent web search results (Redis TTL)
- Run Researcher subtasks in parallel using LangGraph parallel nodes
- Add LangSmith tracing for observability
- Implement rate limiting and exponential backoff on all API calls
- Use checkpointing so interrupted runs can resume from last successful node

IQ2. What's the difference between a "chain" in LangChain and a "graph" in LangGraph? Give an example of when you'd choose each.

Talking points for a strong answer:
- Chain: Linear, always executes the same sequence. Example: question → RAG retrieval → answer. No loops, no branching.
- Graph: Supports conditional edges, loops, parallel execution. Example: research pipeline where Analyst may loop Researcher multiple times.
- Choose chain for: fixed, predictable pipelines; simple RAG; low latency requirements
- Choose graph for: complex workflows; quality-gating; self-correcting systems; anything requiring branching

Section C: Coding Questions

Coding Q1

Task: Define a GraphState TypedDict and a simple Planner node for a multi-agent system that researches a user query.

# Your implementation here

Expected answer:

from typing import TypedDict, List
from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage, HumanMessage
import json

class GraphState(TypedDict):
    query: str
    plan: List[str]
    research_results: List[dict]
    analysis: str
    final_report: str
    iteration_count: int
    approved: bool

llm = ChatOpenAI(model="gpt-4o", temperature=0)

def planner_node(state: GraphState) -> dict:
    system_prompt = """You are a research planner. Given a user query, 
    break it into 3-5 specific, searchable research questions.
    Return a JSON object with a "plan" key containing a list of questions."""

    response = llm.invoke([
        SystemMessage(content=system_prompt),
        HumanMessage(content=state["query"])
    ])

    data = json.loads(response.content)
    return {"plan": data["plan"]}

Coding Q2

Task: Write a routing function should_continue() that decides whether an analyst-approved multi-agent workflow should loop back to the Researcher or proceed to the Writer. Include a max-iteration guard.

# Your implementation here

Expected answer:

def should_continue(state: GraphState) -> str:
    """
    Routing function for the Analyst node's conditional edge.

    Returns:
        "continue" → loop back to Researcher for more investigation
        "done"     → proceed to Writer
    """
    MAX_ITERATIONS = 3

    # Hard stop to prevent infinite loops
    if state["iteration_count"] >= MAX_ITERATIONS:
        return "done"

    # Analyst approved the research quality
    if state.get("approved", False):
        return "done"

    # Research needs more work
    return "continue"
