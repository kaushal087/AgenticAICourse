⚙️ Workflow & Execution Pipeline

End-to-End Execution Flow

Let's trace exactly what happens when a user submits:
"Analyze the impact of generative AI on software engineering careers by 2026"

Step 1: Graph Initialization

# User submits query
initial_state = {
    "query": "Analyze the impact of generative AI on software engineering careers by 2026",
    "plan": [],
    "research_results": [],
    "analysis": "",
    "final_report": "",
    "iteration_count": 0,
    "approved": False
}

# LangGraph compiles and kicks off execution
result = app.invoke(initial_state)

The graph starts at the planner node.

Step 2: Planner Agent

Input: query
Goal: Break the query into 3–5 focused research questions
Prompt strategy: Give the LLM a "Project Manager" persona

System: You are a project planner. Break down the user's research question
        into 3-5 specific, searchable subtasks.

User: "Analyze the impact of generative AI on software engineering careers by 2026"

Expected Output (JSON):
{
  "plan": [
    "What types of software engineering tasks are being automated by AI in 2024-2025?",
    "What do industry surveys say about AI's effect on developer productivity?",
    "What new roles are emerging at the intersection of AI and software engineering?",
    "What are major tech companies saying about AI-driven workforce changes?",
    "What do economists predict about software job growth vs. displacement by 2026?"
  ]
}

State after Planner:

state = {
    "query": "...",
    "plan": ["subtask1", "subtask2", "subtask3", "subtask4", "subtask5"],
    ...
}

Step 3: Researcher Agent

Input: plan (list of subtasks)
Goal: For each subtask, retrieve relevant information using tools
Tools available: web search, RAG retrieval

For each subtask in plan:

Thought: I need to find information about "AI automation of coding tasks"
Action: search_web("AI automation software engineering tasks 2024 2025")
Observation: [Returns web search snippets]

Thought: I also have local documents that may be relevant
Action: rag_retrieve("AI software engineering automation")
Observation: [Returns top-K document chunks]

Thought: I have enough information to summarize this subtask
Final: Compile findings into structured result

State after Researcher:

state = {
    "research_results": [
        {
            "subtask": "What types of software engineering tasks are being automated?",
            "findings": "GitHub Copilot handles 46% of code in repos...",
            "sources": ["github.blog/2024/...", "mckinsey.com/..."]
        },
        # ... more results
    ],
    "iteration_count": 1
}

Step 4: Analyst Agent

Input: research_results
Goal: Synthesize findings, check quality, flag gaps
Logic: If coverage is incomplete, loop back to Researcher

System: You are a rigorous research analyst. Review the findings below.
        - Identify key themes and patterns
        - Flag any conflicting information
        - Assess coverage: are all subtasks well-answered?
        - Return: analysis summary + approved (true/false)

Decision:
- approved = True  → proceed to Writer
- approved = False → loop back to Researcher with additional_queries

Conditional routing function:

def should_continue(state: GraphState) -> str:
    """Route: continue research or proceed to writing?"""
    if state["approved"] or state["iteration_count"] >= 3:
        return "done"      # → Writer
    return "continue"      # → Researcher (loop)

State after Analyst (approved):

state = {
    "analysis": "Key findings: AI is augmenting rather than replacing SWEs...",
    "approved": True
}

Step 5: Writer Agent

Input: query, research_results, analysis
Goal: Produce a well-structured, cited report 

System: You are a technical writer. Produce a well-structured report with:
        - Executive Summary
        - Detailed Findings (one section per subtask)
        - Key Insights
        - Recommendations
        - Sources

Use markdown formatting. Cite sources inline.

State after Writer:

state = {
    "final_report": """
# Impact of Generative AI on Software Engineering Careers by 2026

## Executive Summary
Generative AI is fundamentally reshaping software engineering...

## Detailed Findings

### 1. Automation of Repetitive Tasks
GitHub Copilot data shows 46% of new code is AI-generated...

### 2. Developer Productivity Gains
McKinsey reports 35-45% productivity improvement for augmented developers...

[...continues...]

## Sources
1. GitHub Octoverse 2024 Report
2. McKinsey Technology Trends Report
"""
}

Message Passing Between Agents

All communication happens through the shared GraphState. No direct agent-to-agent calls.

┌──────────┐  writes plan  ┌────────────┐  writes results  ┌──────────┐
│ Planner  │ ──────────▶   │ Researcher │ ──────────────▶  │ Analyst  │
└──────────┘               └────────────┘                  └──────────┘
     ▲                                                           │
     │                                                  approved=True
     │                                                           ▼
     │                                                     ┌──────────┐
     │               writes final_report                   │  Writer  │
     └──────────────────────────────────────────────────   └──────────┘

Why shared state instead of direct message passing?
It makes the system inspectable — you can print the full state at any point. It also makes it easy to resume interrupted runs (checkpointing).

Loop Detection & Guard

To prevent infinite loops:

class GraphState(TypedDict):
    iteration_count: int  # Incremented by Researcher each iteration

def should_continue(state: GraphState) -> str:
    # Hard stop at 3 iterations regardless of approval
    if state["iteration_count"] >= 3:
        return "done"
    if state["approved"]:
        return "done"
    return "continue"

Execution Timeline

t=0s   User submits query
t=1s   Planner completes (fast, single LLM call)
t=5s   Researcher starts subtask 1 (web search + RAG)
t=8s   Researcher completes subtask 1
t=12s  Researcher completes all 5 subtasks
t=15s  Analyst reviews, approves
t=20s  Writer produces final report
t=22s  User sees result

Total: ~20-25 seconds for a 5-subtask research query with GPT-4o.

Streaming Intermediate Results

LangGraph supports streaming node outputs to the UI in real time:

for event in app.stream(initial_state):
    for node_name, node_output in event.items():
        print(f"\n🔄 [{node_name.upper()}] completed")
        if "plan" in node_output:
            print(f"   Plan: {node_output['plan']}")
        if "final_report" in node_output:
            print(f"\n📄 FINAL REPORT:\n{node_output['final_report']}")
