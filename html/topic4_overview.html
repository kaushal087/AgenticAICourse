A multi-agent system is an architecture where multiple specialized AI agents collaborate, each handling a distinct role, to solve problems that are too complex for any single agent.

Think of it like a software engineering team:
- A Planner breaks down a big task into subtasks (like a Tech Lead)
- A Researcher gathers the needed information (like a Senior Dev doing discovery)
- An Analyst evaluates the findings (like a Code Reviewer)
- A Writer produces the final deliverable (like the Dev writing the PR)

Why This Matters

The Problem with Single Agents

Single LLM calls have hard limits:
- Context window exhaustion on long tasks
- No specialization — one agent does everything poorly
- No parallelism — sequential bottlenecks
- No error recovery — one failure breaks the whole pipeline

The Multi-Agent Solution

Challenge Multi-Agent Fix
Long complex tasks Divide across specialized agents
Hallucination Critic/Validator agent checks outputs
Context limits Each agent gets focused, relevant context
Reliability Retry logic per node in the graph

Where Multi-Agent Systems Are Used in Production

1. Autonomous Research Assistants (e.g., Perplexity, Elicit)

Multiple agents search, summarize, and cross-reference documents.

2. Software Engineering Agents (e.g., Devin, GitHub Copilot Workspace)

Planner → Coder → Tester → Reviewer agents collaborate on code changes.

3. Customer Support Automation (e.g., Salesforce Einstein)

Intent classifier → Knowledge retriever → Response drafter → Escalation agent.

4. Financial Analysis Pipelines (e.g., Bloomberg AI)

Data fetcher → Quantitative analyst → Risk assessor → Report generator.

5. Supply Chain Optimization (e.g., enterprise logistics platforms)

Demand forecaster → Inventory optimizer → Shipping coordinator agents.

What We're Building

In this session, we'll build a Multi-Agent Research Assistant using LangGraph.

User asks: "Analyze the impact of AI on software engineering jobs by 2026."

System response:
1. PlannerAgent → decomposes into 4 sub-questions
2. ResearcherAgent → retrieves relevant documents for each
3. AnalystAgent → synthesizes findings & identifies patterns
4. WriterAgent → produces a structured, cited report
